# SprintTracker – Athlete Training and Performance Tracker

# Problem Definition
Many athletes track workouts and performance using notebooks, spreadsheets, or different apps that do not connect with each other. This makes it difficult to see progress over time or understand how training is affecting performance.
SprintTracker is a web application that allows athletes to log workouts, record race results, and view their training progress in one place. The goal of the system is to organize training data and provide simple analytics so users can see trends in their performance.

The intended users are athletes (especially track and field athletes) and coaches who want an easier way to track workouts and monitor improvement.

# Scope and Feasibility

This project focuses on building a full-stack web application that allows users to store and analyze training data. The core features will include user accounts, workout logging, performance tracking, and charts that show training trends over time.
The scope is realistic for one semester because the system is centered around structured data management and visualization rather than highly complex systems. The project will mainly involve building APIs, connecting a frontend interface to a database, and implementing basic analytics for training progress.

Since this project is being completed solo, the feature set will focus on core functionality rather than advanced features like real-time systems or machine learning.


# Technology Stack

# Frontend
- React
- Tailwind CSS
- Chart.js

React will be used to build the user interface. Tailwind CSS will help with styling and layout. Chart.js will be used to display charts that visualize training data and progress.

### Backend
- Node.js
- Express.js

Node.js with Express will handle the server logic and API endpoints that allow the frontend to communicate with the database.

### Database
- PostgreSQL

PostgreSQL will store user accounts, workout logs, and performance data in a structured relational database.

### Version Control
- Git and GitHub will be used to manage the project and track development progress.


## Project Plan and Timeline

The project will be developed in phases throughout the semester.

### Phase 1 – Project Setup (Weeks 1–2)
- Create GitHub repository

### Phase 2 – Backend Development (Weeks 3–5)

- User testing and feedback

### Phase 6 – Finalization (Weeks 14–15)

- Complete documentation
- Prepare project presentation
