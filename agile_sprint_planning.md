# Agile Sprint Planning – SprinterTracker

## 1. Product Vision Statement

SprintTracker is a web application designed for athletes who want a simple way to track workouts and monitor performance over time. Many athletes currently track training in notebooks, spreadsheets, or different apps that do not work together. Because of this, it can be difficult to see progress, analyze training patterns, or understand how workouts are affecting performance.
SprintTracker  solves this problem by providing a centralized platform where athletes can log workouts, record race performances, and visualize their progress through charts and summaries. The goal is to help athletes better understand their training and make more informed decisions about their performance and improvement.

## 3. User Stories

As an athlete, I want to create an account so that I can securely store my training data.

As an athlete, I want to log workouts so that I can track the training I have completed.

As an athlete, I want to record race performances so that I can monitor my improvement over time.

As an athlete, I want to view my workout history so that I can review past training sessions.

As an athlete, I want to see charts of my training data so that I can understand trends in my performance.

## 4. Sprint Plan (Sprint 1)

**Sprint Duration:**  
2 weeks

**Sprint Goal:**  
Set up the project structure and implement the core backend functionality needed to store workout data.

**Sprint Backlog Items**

- Project Setup  
- Database Setup  
- User Authentication  
- Workout Logging Feature  

**Justification**

These tasks were selected first because they create the foundation of the application. Before building the user interface or advanced analytics, the system needs a working backend structure, database, and authentication system so users can store and manage their data.

This project is being completed solo

**Project Coordinator**  
Responsible for organizing tasks, managing sprint progress, and maintaining the project timeline.

**Product Owner role**  
Responsible for defining project priorities and ensuring the application solves the intended problem.

**Developer / Implementation role**  
Responsible for writing the code, building the frontend and backend, managing the database, and implementing system features.

Even though the project is being developed individually, feedback from external users and the instructor will be incorporated to improve the system.

---

## 6. Reflection

One challenge of using Agile for this project is working individually instead of in a group. In a normal Agile environment, team members review each other's work and provide quick feedback and quick help.

To address this challenge, I plan to simulate Agile feedback by asking external users to test the application and provide input on usability and functionality. I will also incorporate feedback from the instructor throughout the project.

Agile practices will help reduce project failure by breaking development into smaller stages instead of trying to build the entire system at once. Working in sprints allows problems to be identified earlier and makes it easier to adjust the project if needed.
